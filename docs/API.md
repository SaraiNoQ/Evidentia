# API Reference

This document describes the API implemented in the current FastAPI backend. It reflects the actual Phase 1 implementation: PDF upload, Paper IR parsing, local deterministic audit, issue listing, evidence lookup, and JSON trace export.

Default local base URL:

```text
http://127.0.0.1:8000
```

## 1. Status

Implemented:

- `GET /health`
- `POST /api/jobs`
- `GET /api/jobs/{job_id}`
- `GET /api/jobs/{job_id}/trace`
- `GET /api/jobs/{job_id}/issues`
- `GET /api/jobs/{job_id}/issues/{issue_id}/evidence`

Placeholder:

- `GET /api/jobs/{job_id}/report?format=markdown`

Not implemented yet:

- Ask the Reviewer.
- Revision comparison.
- PAT-style Markdown/PDF report generation.
- External academic retrieval.
- PostgreSQL/Redis backed persistence.

## 2. Response Envelope

All successful JSON responses use:

```json
{
  "data": {},
  "meta": {
    "request_id": "req_local"
  }
}
```

All error responses use:

```json
{
  "error": {
    "code": "error_code",
    "message": "Human-readable message.",
    "details": {}
  },
  "meta": {
    "request_id": "req_local"
  }
}
```

`request_id` comes from the `x-request-id` request header. If the header is absent, the backend uses `req_local`.

## 3. Core Types

### JobConfig

Used by `POST /api/jobs` as a JSON string in multipart field `config_json`.

```json
{
  "venue": null,
  "review_mode": "local_full_audit",
  "parser_provider": "pymupdf",
  "external_retrieval_enabled": false,
  "max_pages": null,
  "token_budget": null
}
```

Allowed `review_mode` values:

- `quick_audit`
- `local_full_audit`
- `full_review`

Current parser support:

- `pymupdf`: implemented.
- `mineru`: reserved placeholder, returns parse failure if selected.
- `grobid`: reserved placeholder, returns parse failure if selected.

### JobRecord Payload

Returned by job status endpoints:

```json
{
  "job_id": "job_...",
  "status": "completed",
  "warnings": [],
  "error": null,
  "upload_path": "var/uploads/job_.../paper.pdf",
  "paper_ir_path": "var/parsed/job_.../paper_ir.json",
  "trace_path": "var/parsed/job_.../trace.json",
  "created_at": "2026-05-09T00:00:00+00:00",
  "updated_at": "2026-05-09T00:00:00+00:00"
}
```

Current status values:

- `created`
- `parsing`
- `completed`
- `failed`

### EvidenceAnchor

Evidence anchors point to manuscript chunks or artifacts:

```json
{
  "evidence_id": "ev_...",
  "source_type": "paper",
  "source_id": "chunk_...",
  "page": 1,
  "section": "Introduction",
  "line_span": null,
  "artifact_id": null,
  "quote": "Paper evidence quote.",
  "url": null,
  "confidence": 0.75,
  "evidence_level": "B"
}
```

### Issue

Issues are generated by the local audit loop:

```json
{
  "issue_id": "issue_...",
  "title": "Reproducibility details appear incomplete",
  "severity": "possible",
  "dimension": "reproducibility",
  "description": "The local audit did not find explicit mentions of ...",
  "affected_claim_ids": [],
  "evidence": [],
  "counter_evidence": [],
  "missing_evidence": [],
  "recommended_fix": "Add a reproducibility checklist ...",
  "confidence": 0.72,
  "verified_by": ["meta_reviewer"],
  "gate_status": []
}
```

Severity values:

- `fatal`
- `major`
- `moderate`
- `minor`
- `possible`

### PaperTrace

Trace export contains:

- job metadata.
- parser provider.
- job config.
- `PaperDocument` with chunks, artifacts, references and warnings.
- generated file paths.
- local audit summary.
- claims.
- question nodes.
- evidence answers.
- issues.
- agent runs.
- retrieval hits.

## 4. Endpoints

### GET /health

Checks whether the API process is running.

Request:

```http
GET /health
```

Success:

```json
{
  "data": {
    "status": "ok"
  },
  "meta": {
    "request_id": "req_local"
  }
}
```

### POST /api/jobs

Uploads a PDF and creates a job. Current implementation parses synchronously and runs local audit for `quick_audit` and `local_full_audit`.

Request:

```http
POST /api/jobs
Content-Type: multipart/form-data
```

Multipart fields:

- `file`: PDF file.
- `config_json`: optional JSON string matching `JobConfig`.

Example `config_json`:

```json
{"review_mode":"local_full_audit","parser_provider":"pymupdf"}
```

Success:

```json
{
  "data": {
    "job_id": "job_...",
    "status": "completed",
    "warnings": [],
    "error": null,
    "upload_path": "var/uploads/job_.../paper.pdf",
    "paper_ir_path": "var/parsed/job_.../paper_ir.json",
    "trace_path": "var/parsed/job_.../trace.json",
    "created_at": "2026-05-09T00:00:00+00:00",
    "updated_at": "2026-05-09T00:00:00+00:00"
  },
  "meta": {
    "request_id": "req_local"
  }
}
```

Errors:

- `415 unsupported_file_type`: uploaded file is not a PDF.
- `422 invalid_job_config`: `config_json` is malformed or violates schema.
- `422 parse_failed`: parser/provider failed.

### GET /api/jobs/{job_id}/trace

Exports the full JSON trace for Paper IR and local audit output.

Request:

```http
GET /api/jobs/job_123/trace
```

Success:

```json
{
  "data": {
    "schema_version": "paper_ir.v0.1",
    "job": {},
    "parser_provider": "pymupdf",
    "job_config": {},
    "paper_document": {},
    "generated_files": {},
    "warnings": [],
    "summary": {},
    "claims": [],
    "questions": [],
    "evidence_answers": [],
    "issues": [],
    "agent_runs": [],
    "retrieval_hits": []
  },
  "meta": {
    "request_id": "req_local"
  }
}
```

Errors:

- `404 job_not_found`
- `409 report_not_ready`: trace is not written yet.

Request:


```json
{
  "data": {
    "items": [],

    "has_more": false
  },
  "meta": {
    "request_id": "req_local"
  }

```

Errors:

- `404 job_not_found`

```json
  }

```

## 5. Bruno Collection

The Bruno collection is stored at:

```text
bruno/AI Reviewer Agent
```

Open it from Bruno Desktop with `Open Collection`, then select the `Local` environment.

Default variables:

- `base_url`: `http://127.0.0.1:8000`
- `sample_pdf_path`: `/absolute/path/to/paper.pdf`
- `job_id`: empty until a create job request runs.
- `issue_id`: empty until an issue list request runs.

The create-job requests include post-response scripts that store `job_id`. The list-issues request stores the first returned `issue_id` when available.
